# getinside Handbook — Project Context

## Project Overview

**getinside Handbook** is a bilingual (French/English) documentation site built with **VitePress** for the getinside Retail Media SaaS platform. It serves as the official knowledge base for two audiences:

- **Advertisers (Brands)**: Guides on running campaigns via parcel inserts, dedicated emails, display email, and social ads
- **Publishers/Retailers (E-retailers)**: Guides on monetizing shipments, managing audiences, receiving payouts, and growing revenue

**Live Site**: [https://getinside-ops.github.io/handbook/](https://getinside-ops.github.io/handbook/)

**Tech Stack**:
- **Engine**: VitePress (Vue-based static site generator)
- **Theme**: Custom getinside theme with Vue components
- **Hosting**: GitHub Pages
- **Language**: Markdown (`.md`) with bilingual content (FR + EN)
- **Styling**: Custom CSS with brand tokens (mint/teal accent colors)

## Building and Running

### Prerequisites
```bash
npm install
npx playwright install chromium  # For OG image generation
```

### Commands
```bash
npm run docs:dev      # Local dev server with hot-reload
npm run docs:build    # Build to .vitepress/dist/ + generate sitemap.xml
npm run docs:preview  # Preview production build locally
npm run generate:og   # Regenerate OpenGraph image (1200×630)
```

### Build Output
- Static files generated to `.vitepress/dist/`
- `sitemap.xml` auto-generated post-build with all pages
- Deployed to GitHub Pages at `/handbook/` base path

## Content Conventions

### Frontmatter Requirements

**Every page MUST include YAML frontmatter**:
```yaml
---
title: SEO-optimized page title (50-60 chars)
description: Meta description (120-160 chars, unique per page)
keywords:
  - keyword1
  - keyword2
  - keyword3
image: /images/og-image.png  # Optional, overrides default
---
```

**Every page MUST have an explicit H1** immediately after frontmatter (VitePress doesn't auto-render `frontmatter.title`):
```markdown
---
title: Page Title
---

# Page Title (visible H1)

Content starts here...
```

**Exception**: Root `index.md` uses `layout: home` with `hero` and `features` blocks.

### VitePress Components

**Callout containers** (styled via `style.css`):
```markdown
::: tip
Positive highlight or key insight
:::

::: info
Informational callout
:::

::: warning
Caution or attention point
:::

::: danger
Critical alert or warning
:::

::: details Summary
Collapsible content (FAQ style)
:::
```

**Custom HTML components** (styled in `style.css`):

- `<div class="gi-step">` — Numbered process steps
- `<div class="gi-value-grid">` — Metric/comparison cards
- `<div class="gi-format-grid">` — Format showcase with linkable cards
- `<div class="gi-quick-links">` — Action button containers

### Tables
All markdown tables render full-width by default (`.vp-doc table { width: 100% }`).

## SEO & Metadata

### Automatic Injection

The theme (`theme/index.ts`) and config (`config.ts` via `transformHead`) automatically inject:

- **OpenGraph tags**: `og:title`, `og:description`, `og:image`, `og:url`
- **Schema.org JSON-LD**: WebPage, Organization, BreadcrumbList
- **Canonical URLs**: Based on page path
- **Hreflang alternates**: FR ↔ EN cross-linking
- **Meta description**: From frontmatter
- **AI/LLM tags**: `citation_title`, `dc.title`, `ChatGPT-content`

### Sitemap

Generated by `scripts/generate-sitemap.js` post-build:
- All `.md` files included (except `404.md`)
- Priority assigned by section (homepage=1.0, start-here=0.9, advertisers/publishers=0.8)
- `lastmod` auto-set from file modification time

### Keywords Reference

See `seo-keywords.md` for comprehensive keyword index covering:
- Retail media terminology
- E-commerce monetization
- Advertising formats
- Trust transfer & endorsement concepts
- First-party data & targeting

## Content Guidelines

### Writing for Humans & LLMs

1. **Write for humans first**, SEO second
2. Integrate keywords **naturally** (no keyword stuffing)
3. Link to **semantically related** pages
4. Use **descriptive headings** (H2, H3)
5. Structure with **lists** and **tables** for scannability

### LLM Optimization

Include for maximum LLM citation potential:
- **Statistics & data points** (e.g., "65-70% net margin")
- **Case studies** with concrete results
- **Comparative data** (tables, benchmarks)
- **Unique insights** (getinside differentiators)
- **Conversational keywords** (questions LLMs might ask)

### Internal Linking

Link to hub pages for authority:
- `/start-here/` → Advertisers + Publishers overview
- `/advertisers/` → Each ad format page
- `/publishers/` → Each monetization lever
- `/faq/` → All troubleshooting sections

## Development Workflow

### Adding a New Page

1. Create `.md` file in appropriate `docs/<section>/` folder
2. Add frontmatter with title, description, keywords
3. Add H1 heading after frontmatter
4. Write content (min 300 words, structured with H2/H3)
5. Add to sidebar in `.vitepress/config.ts`
6. Create English translation in `docs/en/` (maintain parity)
7. Update `seo-keywords.md` if new topics introduced

### Content Audit Commands

Check for missing H1:
```bash
for f in $(find docs -name "*.md" -not -path "*/superpowers/*"); do
  if ! grep -q "^# " "$f" && ! grep -q "^layout:" "$f"; then
    echo "MISSING H1: $f"
  fi
done
```

### Before Committing

- [ ] Frontmatter complete (title, description, keywords)
- [ ] H1 heading present
- [ ] Min 300 words with natural keyword integration
- [ ] 2-3 internal links to related pages
- [ ] English translation created/updated
- [ ] Added to sidebar in `config.ts`
- [ ] Build succeeds: `npm run docs:build`

## Troubleshooting

### Build Fails
- Check for missing H1 headings
- Verify frontmatter syntax (proper YAML)
- Ensure all internal links resolve

### SEO Not Working
- Verify frontmatter has `title` and `description`
- Check `transformHead` in `config.ts` is injecting tags
- Confirm `theme/index.ts` is updating metadata on route change

### Sitemap Missing Pages
- Ensure page is not in `EXCLUDED_PAGES` array
- Check file is in `docs/` or `docs/en/`
- Re-run `npm run docs:build`
