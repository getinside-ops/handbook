## Paper formats

| Format | Finished (cut) | File (with bleed) | Safe zone |
| :--- | :--- | :--- | :--- |
| **A5** | 148 × 210 mm | **152 × 214 mm** | 3 mm internal |
| **A6** | 105 × 148 mm | **109 × 152 mm** | 3 mm internal |
| **Card** | 105 × 148 mm | **109 × 152 mm** | See template |

:::warning Bleed: 2 mm mandatory
The material (image or colored background) must extend **2 mm outside** the finished format. Without bleed, a white border appears after cutting.
:::

:::tip Safe zone: 3 mm
No text, logo, or important element should be less than **3 mm from the inner edge**. Anything too close to the edge risks being cropped.
:::

## Files & colorimetry

| Criterion | Specification |
| :--- | :--- |
| **File format** | PDF/X vector (ISO prepress standard) — preferred. JPG or PNG accepted. |
| **Resolution** | **300 DPI minimum** |
| **Raster dimensions** | A6: 1,287 × 1,795 px min · A5: 1,795 × 2,528 px min |
| **Color mode** | **CMYK mandatory** (no RGB) |
| **ICC Profile** | Coated FOGRA39 |
| **Ink coverage** | **Maximum 300%** |

:::tip Deep black
For a rich black on solid areas (large black backgrounds): use **C30% M30% Y30% K100%**.
For fine texts and small characters: use **K100% only** to avoid color shifting during printing.
:::

## Paper

| Criterion | Value |
| :--- | :--- |
| **Weight** | 250 to 300 g/m² |
| **Type** | Matte coated or semi-matte coated |
| **Certification** | FSC or PEFC |
| **Lamination** | Prohibited |

:::warning Lamination prohibited
Lamination (matte or glossy plastic coating) is prohibited for two reasons:
1. Laminated paper cannot be recycled through standard paper recycling.
2. Lamination reduces QR code readability by smartphone cameras.

To protect the print, use **machine varnish** instead — it is recyclable and does not affect QR code scanning.
:::

## getinside QR Code

:::danger Proprietary QR code mandatory
You **cannot** use a QR code generated by yourself. You must integrate the **`.svg`** file provided by the getinside platform.

This QR code certifies distribution, enables real-time monitoring, and activates your performance dashboard.
:::

**Minimum size: 25 × 25 mm** — below this, the QR code is not readable by all smartphones.

The QR code is available for download in your advertiser dashboard, in the campaign section, once your campaign proposal has been approved by the retailer.

## Promo codes

Use a specific promo code per retailer to precisely trace the origin of your conversions.

- **One code per retailer** — include the retailer brand name or abbreviation (reinforces perceived synergy)
- **Short and memorable** — no special characters (`-` `_` `,` `.` `;` `:` should be avoided)
- **Minimum validity: 3 months** — recipients often keep flyers for several weeks before converting
- **Unique-code campaigns** — provide 1 `.csv` file per audience to getinside

## Key checks

:::warning Before submitting your file
- **Proofread** your text, verify specs, and display overprints in your software
- **Don't mix QR codes** — each retailer has their own, distinct code
- **Deep black** (large solid areas): C30% M30% Y30% K100%
- **Fine text**: K100% only to avoid color registration issues during printing
- **Ink coverage**: the sum C+M+Y+K must not exceed **300%** per point
- **Resolution**: all images must be at **300 DPI minimum**
:::

---

## Proof (BAT) checklist

Check before each submission:

1. Dimensions include **+2 mm bleed** and respect the **-3 mm safe zone**.
2. File is in **CMYK (FOGRA39)**, resolution ≥ **300 DPI**, ink coverage ≤ **300%**.
3. getinside QR code (`.svg`) is present and at least **25 × 25 mm**.
4. Universal legal notices are included: advertiser ID, Triman, getinside logo, T&Cs.
5. [Sector-specific mandatory notices](./legal) (if applicable) are present and legible.
